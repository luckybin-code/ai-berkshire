# 딥 컴퍼니 시리즈: 8편의 장문으로 한 기업을 파헤치다

$ARGUMENTS 에 대해 8편짜리 심층 장문 시리즈를 작성하고, 공식 계정/영상 채널 등 공개 채널에 발행한다. **핵심 경쟁력은 "잘 쓰는 것"이 아니라 "제대로 고치는 것"이다 — 시중 경제 기사의 99%가 이 skill의 팩트체크 기준을 위반하고 있다.**

참고 샘플: `reports/텐센트/《텐센트 제대로 알기》/`

---

## 1. 사용 시나리오

사용자가 한 기업에 대해 "교과서급" 심층 연구를 **시리즈 장문** 형태로 공개 발행하고자 할 때 사용한다. 일반 리포트 한 편과는 다르다.
- 8편, 약 12만 자 분량, 인식 리셋에서 의사결정 프레임까지 완결된 흐름
- 각 편은 독립적으로도 완결(단독 공유 가능)되지만, 하나의 밸류에이션/경영진/가격 판단 체계가 전체를 관통
- "이 기업을 제대로 이해하는 데 90분을 쓸 의향이 있는" 독자를 위한 글이지, 증권사 고객을 위한 글이 아님

**이 skill을 쓰기에 적합하지 않은 시나리오**: 단발성 리포트, 분기 실적 코멘트, 업종 리서치 — 이런 경우는 `/investment-research`, `/earnings-review`, `/industry-research`를 사용한다.

## 2. 시리즈 목차 템플릿(8편)

| # | 편명 템플릿 | 핵심 질문 | 분량 |
|---|---------|---------|------|
| 01 | 당신이 X를 안다고 생각했지만, 사실은 몰랐다 | 인식 리셋: 흔한 오해 3가지를 깬다 | 4,000-5,000자 |
| 02 | X의 해자 — `<사업 본질 한 줄 요약>` | 해자가 깊은가, 향후 5/10년 뒤에도 남아 있을 것인가 | 6,000-8,000자 |
| 03 | X의 최대 이익 엔진 — `<가장 돈이 되는 사업>` | 본업이 무엇이고, 왜 지속 가능한가 | 6,000-8,000자 |
| 04 | X의 장부 뒤에 숨겨진 또 하나의 회사 — `<숨겨진 자산>` | 투자 포트폴리오 / 자회사 / 숨겨진 가치 | 8,000-10,000자 |
| 05 | AI(또는 현재의 지배적 내러티브) 시대, X는 승자인가 패자인가 | 시대 변수: 사업별로 AI 영향을 분해 | 8,000-10,000자 |
| 06 | 버핏 방식으로 X의 재무제표를 뜯어보다 | 재무 심층 분석: 매출총이익률/FCF/ROE/SBC | 8,000-10,000자 |
| 07 | `<경영진 어록>` — X의 경영진은 신뢰할 만한가 | 자본배분 규율 + 진실성 검증 + 후계자 문제 | 8,000-10,000자 |
| 08 | 얼마면 살 만하고, 어떤 신호가 오면 반드시 팔아야 하는가(시리즈 대미) | DCF 3개 시나리오 + 레드라인 체크리스트 + 포지션 프레임 | 10,000-12,000자 |

발행하지 않는 `00-시리즈 소개.md`를 목차 색인으로 추가한다.

## 3. 작성 스타일 규정

### 어조

- **직접적, 예리하게, 군더더기 없이** — 첫 문장부터 숫자나 반상식적 결론을 던진다
- **가치투자 프레임** — 버핏/멍거/돤융핑/리루의 관점을 곳곳에 배치(다만 명언을 남발하지 않는다)
- **입장을 전제하지 않는다** — 먼저 데이터를 제시하고, 논리를 전개한 뒤, 마지막에 결론을 낸다
- **양면을 모두 제시** — 모든 핵심 판단에는 "하지만 다른 한편으로는..."이라는 반론을 첨부한다
- **공식 계정 감각** — 첫 18-20자만으로도 독립적으로 성립해야 한다(모바일 미리보기 기준)

### 금지 표현

| 금지어 | 이유 | 대체 표현 |
|------|------|------|
| 명백히 / 필연적으로 / 반드시 | 주관적 단정 | 데이터에 따르면 / 증거가 보여주듯 |
| 나는 생각한다 / 내 생각에는 | 주관적 어투 | 삭제하거나 "이 프레임에 따르면"으로 대체 |
| 교과서급 / 신의 한 수 | 트래픽 목적의 과찬 | 구체적 사실을 서술 |
| 심각하게 어긋난다 / 심각하게 저평가되었다 | 강한 주관어 | 구체적인 할인율 수치를 제시 |
| 완벽하다 / 흠잡을 데 없다 | 일방적 판단 | 반대 관점을 함께 서술 |

### 제목 스타일

- **대비되는 숫자**나 **반(反)컨센서스 결론**으로 후킹한다("15년간 7번 도전 전부 실패", "연봉 4,292만 원, 이익의 0.0017%")
- 부제는 중립적으로, 내용을 개괄한다("——`<본질 판단>`")
- **트래픽성 비유는 반드시 피한다**: "제2의 버핏", "중국판 X", "레전드급" 같은 표현은 일절 쓰지 않는다
- 전문 독자에게 익숙한 용어를 쓴다(인물명보다 "버크셔 해서웨이"처럼 회사명을 우선한다)

## 4. 엄격한 팩트체크 체크리스트(핵심 경쟁력)

### 작성 전부터 경계해야 할 "가짜 정밀함" 함정

1. **확률가중 기댓값**: `30% × A + 50% × B + 20% × C = 기대 +X%` 이런 계산은 거의 다 쓰레기다 — 확률 배분은 순전히 주관적이며, 독자에게 잘못된 정밀함의 착각을 준다. **시나리오 + 트리거 조건만 나열하고, 가중 기댓값은 계산하지 않는다.**
2. **제3자 추정 MAU/점유율**: QuestMobile/치마이(七麦)/CBNData 등은 기준이 크게 달라(같은 시점에도 2-3배 차이가 날 수 있다). **가장 신뢰할 수 있는 두 소스만 비교 앵커로 쓰고, 나머지는 정성적으로 서술한다.**
3. **과거 증가율의 선형 외삽**: `2025년 +33% × 5년 복리 → 2030년 X` 는 금융 문맹식 예측이다. **시나리오 가정 + 고/저 구간 + 이는 약속이 아님을 명시한다.**
4. **비공개 지분율**: 바이트댄스, Halti류의 비상장 기업 지분은 **공식적으로 공개된 적이 없다**. **구간으로 제시하고 "알 수 없음"으로 표기한다.**
5. **강한 인과 귀속**: 경쟁사의 실패 = X 때문이라는 식의 단정. 여러 원인을 모두 나열하고, **본문에서는 단일 귀속을 하지 않는다.**

### 수정 시 반드시 거쳐야 할 7가지 점검

```
□ 1. 편 간 숫자 일관성: 시가총액, Non-IFRS 순이익, 핵심 지분율이 전 시리즈에서 일치하는가
□ 2. 기준 표기: Non-IFRS / GAAP / Non-IFRS-SBC / FCF 중 어느 것을 쓰는지 전체 글에서 명확한가
□ 3. 중복 합산 스캔: 이미 연결된 자회사가 "투자 포트폴리오"에 다시 들어가지 않았는가, SOTP가 이중 계산되지 않았는가
□ 4. 횡비교 공정성: "본업 PE(현금+포트폴리오 제외)" vs "경쟁사 PE(제외 안 함)" 같은 불공정 비교가 없는가
□ 5. 확률가중 전면 삭제: 위 항목 참고
□ 6. 단정적 표현 전면 완화: "명백히|필연적으로|심각하게|교과서급|완벽하다" grep 검색
□ 7. 제3자 데이터 출처 표기: 재무제표 외 모든 데이터 뒤에 "(출처: X)" 표기
```

### 모델이 흔히 저지르는 오류

작성 전에 **이미 알려진 확실한 오류 위험**을 먼저 나열한다.
- 과거 수익 배수: 반드시 누적 투입 기준으로 계산(예: Riot는 33배이지 58배가 아님)
- 지분율: 반드시 최신 Futu(富途)/재무제표 기준으로 확인(예: 텐센트가 보유한 메이투안 지분은 1.5%이지 6.4%가 아님)
- "배당 분배" 회계 처리: 처분 간주 손익은 IFRIC 17에 따라 선언일에 인식(예: 징둥은 2021년, 메이투안은 2022년이지만 금액은 작음)
- 총주식수는 반등할 수 있음: SBC가 연초에 집중 부여되면 주식수가 단기적으로 증가할 수 있음

## 5. 실행 절차

### 1단계: 조사(01-02편 작성 전 완료)

1. 회사의 최근 5년치 연차보고서, 최신 분기보고서를 읽는다
2. 최소 3편의 독립적인 매도측 리포트를 읽는다(컨센서스 + 반컨센서스를 찾는다)
3. `/investment-team` 또는 `/investment-research`를 사용해 내부 리서치 초안을 먼저 생성한다
4. 사용자와 8편의 핵심 논지를 확인한다(다 쓴 뒤 방향이 틀렸다는 것을 발견하는 것을 방지)

### 2단계: 집필(01→08 순서로 작성, 건너뛰지 않는다)

- 각 편을 다 쓰면 먼저 `reports/{회사명}/《{회사명} 제대로 알기》/0X-XX.md`에 저장한다
- 즉시 GitHub에 푸시하지 않는다 — 사용자 검토를 기다린다
- 사용자가 수정 의견을 준 뒤 수정한다
- 수정이 끝난 뒤에야 git push한다

### 3단계: 편 간 일관성 스캔(8편 전부 작성 완료 후)

Explore 에이전트를 파견해 8편을 병렬로 스캔하며 아래를 점검한다.
1. 동일 숫자(시가총액, 순이익, 지분율)가 편마다 일치하는가
2. 동일 용어(FBS, SBC, Non-IFRS)가 처음 등장할 때 설명되었는가
3. 인용 관계: 02편에서 "06편 참조"라고 했으면 실제로 대응하는가
4. 요점 정리와 본문의 숫자가 일치하는가

### 4단계: 발행 전 최종 점검

```bash
# 푸시 전 반드시 로컬에서 한 번 grep한다(ai-berkshire 프라이버시 규정에 따름)
grep -r "linxuan\|/Users/\|<사용자 회사 별칭>" reports/ | head
```

이상이 없음을 확인한 뒤에야 `git pull --rebase && git commit && git push`를 실행한다.

## 6. 수정 의견 처리 절차

사용자가 수정 의견을 줄 때는 다음 순서로 처리한다.

### 1. 먼저 사실을 확인한다(바로 고치지 않는다)

사용자가 "X 데이터가 틀렸다"고 하면, 먼저 Bash/Read로 원본 데이터를 찾아 교차 검증한다.
- ai-berkshire 프로젝트 내 같은 회사의 earnings/실적 리포트를 확인한다
- Futu(富途)/공식 공시를 확인한다
- "사용자가 말한 데이터 vs 내가 확인한 데이터 vs 이전에 썼던 데이터" 3자 비교를 제시한다

### 2. 수정 등급을 판단한다

| 등급 | 유형 | 처리 |
|------|------|------|
| 🔥 명확한 오류 | 숫자 오류, 귀속 오류, 기준 오류 | 반드시 수정, 망설일 필요 없음 |
| ⚠️ 주관화 | 강한 주관어, 단정적 표현, 트래픽성 비유 | 완화하거나 삭제 |
| 🔬 세밀도 | 출처 표기, 기준 세분화 | 우선순위 낮음, 가독성과 균형 고려 |
| ❓ 신뢰 불가 | 제3자 추정치 간 차이가 큼 | **고치기보다 삭제가 더 안전함**(사용자 명시적 지시) |

### 3. 수정 후 연쇄 점검

한 곳을 고칠 때 먼저 "이 숫자/개념을 참조하는 다른 곳은 어디인가"를 생각한다. 예:
- 총 시가총액을 고쳤다면 → 전 시리즈에서 PE / 본업 PE / 할인율 / FCF Yield도 연쇄 수정
- 지분율을 고쳤다면 → TOP 10 순위 + 과거 지분표 + 매도 목록도 수정
- 용어 기준을 고쳤다면 → 최초 정의 + 이후 인용 + 요점 정리도 수정

### 4. 푸시 후 즉시 보고

```
푸시 성공(commit hash).
[N]건 수정 요약(표 첨부):
- 무엇을 고쳤는가
- 연쇄로 무엇을 고쳤는가
- 아직 고치지 않은 것은 무엇인가

다음 지시를 기다립니다.
```

## 7. 이 skill이 하지 않는 것

- **독자를 대신해 투자 결정을 내리지 않는다** — 모든 편의 말미에 "투자 권유가 아님"을 명시
- **주가를 예측하지 않는다** — "시나리오 + 트리거 조건"만 제시
- **"기대 연환산 수익률" 가중값을 계산하지 않는다** — 주관적 확률 배분은 독자를 오도할 수 있음
- **"X 거물도 보유 중"이라고 쓰지 않는다** — 타인의 보유를 근거로 자신의 판단을 정당화하는 것은 가치투자 정신에 반한다
- **8편을 무리하게 다 채우지 않는다** — 어떤 편이 충분한 독립 콘텐츠가 없으면(예: 어떤 회사의 경영진이 특별할 것이 없는 경우) 다른 편에 합치거나 편수를 줄인다

---

## 8. 컴플라이언스와 프라이버시

- 모든 공개 리포트는 **공개 정보만 사용**한다(재무제표, 공식 공시, 증권사 리포트, 저명한 제3자 기관)
- 어떠한 **사용자 개인정보**도 사용하지 않는다(회사 내부 별칭, 내부 메신저, 비공개 보유 정보)
- 푸시 전 반드시 grep으로 `linxuan` / `/Users/` / 사용자 회사 별칭 등 프라이버시 필드를 스캔한다(`~/.claude/projects/-Users-linxuan/memory/feedback_privacy_upload.md` 참고)
- 공개 서명은 사용자의 다중 계층 신원 전략에 따르며, 혼용하지 않는다

## 한 줄 요약

**《X 제대로 알기》 시리즈를 쓰는 핵심 역량은 잘 쓰는 것이 아니라 엄격하게 고치는 것이다** —
경제 장문의 89%는 가짜 정밀 숫자, 주관적 가중 기댓값, 단정적 표현 때문에 실패한다. 이 skill의 존재 이유는 이런 함정들을 전부 표시해내는 것이다. 쓰기 전에는 피하고, 쓰고 난 후에는 깨끗이 정리한다.
