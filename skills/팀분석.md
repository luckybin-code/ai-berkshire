# 투자연구팀: 4개 역할 병렬 분석 프레임워크

$ARGUMENTS 에 대해 팀 단위 투자 리서치 분석을 수행한다. Team 도구를 사용해 실제 멀티 에이전트 병렬 연구팀을 만든다.

## 실행 절차

### 1단계: 팀 구성 제시

사용자에게 아래 팀 구조를 보여주고, 확인을 받은 뒤 시작한다:

| 역할 | 임무 | 분석 프레임워크 |
|------|------|----------|
| **team-lead**(본인) | 총괄 조율, 결론 종합, 최종 보고서 출력 | 4대 거장 종합 프레임워크 |
| **business-analyst** | 비즈니스 모델 & 해자(moat) 분석 | 돤융핑 관점 |
| **financial-analyst** | 재무제표 & 밸류에이션 분석 | 버핏 관점 |
| **산업리서치er** | 산업 구도 & 경쟁 양상 | 멍거 관점 |
| **risk-assessor** | 리스크 평가 & 경영진 평가 | 리루 관점 |

### 1.5단계: AI 리서치 편향 평가

팀을 만들기 전, 먼저 해당 기업의 "AI 리서치 가능성"에 대한 평가를 사용자에게 제시한다:

**정보 풍부도 등급**(리서치 전략을 결정함):
| 등급 | 특징 | 리서치 전략 조정 |
|------|------|------------|
| A등급(정보 풍부) | 상장 오래됨, 증권사 커버리지 넓음 | 팀은 **반대 검증**과 **비합의(non-consensus) 시각**에 집중하여, 시장과 똑같은 "맞는 말이지만 의미 없는" 결론을 내지 않도록 함 |
| B등급(정보 보통) | 상장한 지 얼마 안 됨, 커버리지 제한적 | 각 에이전트의 추정치는 반드시 신뢰도를 표기하고, team-lead가 종합할 때 "데이터 충분도"를 표기 |
| C등급(정보 희소) | 비인기/신규상장/신흥시장 | 팀은 "제1원리 모드"로 전환: 보고서의 완전성을 추구하지 않고, 비즈니스 본질에 관한 핵심 질문 몇 가지에 집중 |

**핵심 유의사항**: 자료가 많다고 확실성이 높은 것은 아니고, 자료가 적다고 확실성이 낮은 것도 아니다. AI가 낼 수 있는 신뢰도 ≠ 투자의 실제 확실성. 확실성은 자료의 양이 아니라 비즈니스 모델 자체에서 나온다.

이 등급 평가 결과를 각 에이전트에게 알려주어 리서치 방식에 반영시킨다.

### 2단계: 팀 생성

TeamCreate로 팀을 생성한다:
- team_name: `{회사명}-research`(영문 소문자, 예: `meituan-research`)
- agent_type: `team-lead`

### 3단계: 4개 태스크 생성

TaskCreate로 아래 4개 태스크를 생성한다(각각 subject, description, activeForm을 모두 포함해야 함):

#### 태스크1: 비즈니스 모델 분석
- subject: `{회사명}의 비즈니스 모델, 해자, 사용자 가치 분석`
- description에 포함할 내용:
  1. 비즈니스 모델의 본질: 핵심 사업 정의, 매출 구조 분해
  2. 플랫폼/제품 플라이휠 효과가 어떻게 작동하는지
  3. 해자 분석: 브랜드/전환비용/네트워크효과/규모의경제/기술장벽을 하나씩 검증
  4. 사용자/고객 가치: 각 이해관계자에게 어떤 고유한 가치를 창출했는지
  5. 사업 포트폴리오와 시너지 효과
  6. 돤융핑의 "좋은 사업" 기준 평가: 차별화, 가격결정력, 지속가능한 경쟁우위
  7. 최신 실적발표, 산업 리포트 등 공개 정보를 검색할 것

#### 태스크2: 재무 및 밸류에이션 분석
- subject: `{회사명}의 재무 데이터, 수익성, 밸류에이션 분석`
- description에 포함할 내용:
  1. 최근 3-5년 매출, 순이익, 영업이익 추세
  2. 수익성 지표: ROE, ROA, 매출총이익률, 영업이익률
  3. 현금흐름 분석: 영업현금흐름, 잉여현금흐름, 자본적지출
  4. 재무상태표 건전성: 현금 보유고, 부채비율, 유동성
  5. 밸류에이션 분석: PER/PSR/PBR/EV 등, 과거 및 동종업계 대비
  6. 안전마진 평가: 내재가치 vs 현재 주가
  7. **금융적 엄밀성 검증(반드시 Bash로 도구를 호출하고, 암산 금지)**:
     - 시가총액 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap --price {가격} --shares {주식수} --reported {보고된 시가총액} --currency {통화}`
     - 밸류에이션 검산: `python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation --price {가격} --eps {EPS} --bvps {주당순자산}`
     - 핵심 데이터 교차검증: `python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate --field {필드} --values '{JSON}' --unit {단위}`
     - 3가지 시나리오 밸류에이션: `python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario --price {가격} --eps {EPS} --shares {주식수(억)} --growth {낙관} {중립} {비관} --pe {낙관PE} {중립PE} {비관PE}`
     - 도구 출력 결과를 검증 기록으로 보고서에 그대로 삽입

#### 태스크3: 산업 및 경쟁 분석
- subject: `{산업} 산업 구도 및 {회사명}의 경쟁 양상 분석`
- description에 포함할 내용:
  1. 산업 규모와 성장성: 시장 규모, 성장률, 침투율
  2. 경쟁 구도: 주요 경쟁사의 시장점유율, 경쟁 전략 비교
  3. 핵심 경쟁자 위협 평가: 주요 경쟁사를 하나씩 분석
  4. 각 세부 영역별 구도
  5. 산업 트렌드: 기술 변화, 정책 영향, 신규 진입자
  6. 산업체인 분석: 상류-중류-하류 가치 배분
  7. 최신 산업 데이터와 경쟁 동향을 검색할 것

#### 태스크4: 리스크 및 경영진 평가
- subject: `{회사명}의 투자 리스크와 경영진 자질 평가`
- description에 포함할 내용:
  1. 경영진 평가: CEO의 능력범위, 정직성, 전략적 안목, 자본배분 능력, 과거 의사결정 품질
  2. 규제 리스크: 현재 및 잠재적 규제 영향
  3. 경쟁 리스크: 각 경쟁사의 위협 정도 평가
  4. 사업 리스크: 신사업 손실, 확장의 불확실성
  5. 거시 리스크: 경기 사이클, 산업 사이클 영향
  6. 지배구조: 지분 구조, 특수관계자 거래, 주주환원 정책
  7. 장기적 확실성: 10년 후 회사는 어떤 모습일까? 무엇이 이 비즈니스 모델을 뒤엎을 수 있는가?
  8. 최신 규제 동향, 경영진 발언 등을 검색할 것

### 4단계: 4개 에이전트 병렬 실행

Task 도구로 4개 에이전트를 동시에 실행한다(**반드시 하나의 메시지 안에서 병렬로 호출**):

각 에이전트의 설정:
- `subagent_type`: `general-purpose`
- `run_in_background`: `true`
- `team_name`: 해당 팀명
- `name`: 해당 역할명(business-analyst / financial-analyst / 산업리서치er / risk-assessor)

각 에이전트의 prompt 템플릿:

```
당신은 {회사명} 투자연구팀의 "{역할 한글명}"이며, {거장명}의 투자 관점에서 {회사명}을 분석하는 역할을 맡는다.

태스크 #{태스크번호}: {태스크 subject} 를 완수하라.

구체적 요구사항:
{태스크 description 내용}

**리서치 방법**:
- WebSearch로 최신 공개 정보(실적발표, 산업 리포트, 뉴스)를 검색할 것
- **재무 데이터는 반드시 두 개의 독립된 출처**에서 확보하며, `skills/financial-data.md` 규정을 따를 것(미국주식: macrotrends+stockanalysis; 홍콩주식: aastocks+macrotrends; 중국A주: 동방재부+주차이자쉰), 두 출처 오차가 1%를 넘으면 반드시 표기
- 데이터 정확성을 확보하고, 핵심 데이터에는 출처를 표기할 것
- 분석은 깊이 있게, 피상적으로 흐르지 않게 할 것

**출력 요구사항**:
- 보고서는 상세해야 하며, Markdown 표로 핵심 데이터를 제시할 것
- 각 분석 차원마다 명확한 결론과 점수를 부여할 것
- 보고서 끝에 해당 차원의 전체 결론을 넣을 것

**완료 후**:
1. TaskUpdate로 태스크 #{태스크번호}를 completed로 표시
2. SendMessage로 전체 분석 보고서를 team-lead에게 전송(type: "message", recipient: "team-lead")
```

### 5단계: 보고서 수신 및 진행 상황 추적

- 사용자에게 실시간 진행 상황표(어떤 에이전트가 완료됐고 어떤 에이전트가 아직 리서치 중인지)를 보여준다
- 보고서를 받을 때마다 진행 상황을 갱신하고, 해당 보고서의 핵심 요점(3-5개)을 보여준다
- 4개 보고서가 모두 도착할 때까지 기다린다

### 6단계: 팀 멤버 종료

모든 보고서를 받으면, 4개 에이전트에게 shutdown_request를 전송한다(SendMessage 사용, type: "shutdown_request").

### 7단계: 최종 보고서 종합

4개 분석 보고서를 종합하여 아래 구조의 최종 보고서를 출력한다:

---

#### 1. 한 줄 결론
> 한 문단(50-100자 정도)으로 투자 가치 여부와 핵심 논리를 요약

#### 2. 4차원 평가 종합표
| 차원 | 프레임워크 | 점수(1-5점) | 핵심 판단 |
|------|------|------------|----------|

종합 점수: X / 5

#### 3. 핵심 데이터 요약
핵심 재무 및 경영 지표 표(최근 2년 비교)

#### 4. 각 차원별 분석 요약
각 차원에서 가장 중요한 발견 3-5개를 발췌

#### 5. 투자 논점(Bull vs Bear)
- 🟢 매수 논리(5-7개)
- 🔴 매도/회피 논리(5-7개)

#### 6. 버핏식 매수 전 체크리스트
| # | 점검항목 | 통과? | 설명 |
핵심 점검항목 10개를 하나씩 평가

#### 7. 최종 투자 제안
- 정성적 판단표(사업 품질/경영진/밸류에이션/타이밍)
- 계층별 대응 제안표(공격형/중립형/보수형 → 제안+가격대)
- 핵심 촉매(비중 확대 신호/축소 신호 각 3-5개)

#### 8. 요약 단락
100-200자 정도의 최종 요약

---

### 8단계: 보고서 저장

완성된 최종 보고서를 `~/{회사명}투자연구보고서_{날짜}.md`(날짜 형식 YYYYMMDD)에 저장한다.

### 9단계: 데이터 샘플 검사(출고 프로세스)

```bash
# Step 1 — 샘플 검사 목록 추출(15% 무작위 샘플링)
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>

# Step 2 — 목록의 각 항목을 신뢰할 수 있는 출처에서 재확인(skills/financial-data.md 참고)

# Step 3 — 출고/반려 판정 출력
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<작성된 JSON>' \
  --report <보고서 파일명>
```

**【출고】** 전부 통과 → 보고서 발행 가능; **【반려】** 미통과 있음 → 수정 후 재검토.

### 10단계: 팀 정리

TeamDelete로 팀 자원을 정리한다.

## 중요 유의사항

1. **4개 에이전트는 반드시 병렬로 실행**——하나의 메시지 안에서 Task 도구를 4번 호출
2. **에이전트는 SendMessage로 보고**——파일 협업이 아니라 메시지 통신 방식
3. **데이터 정확성**——에이전트가 WebSearch로 최신 데이터를 검색하고 핵심 데이터를 교차검증하도록 요구
4. **결론은 명확하게**——매수/관망/회피 제안과 구체적인 가격대 제시를 회피하지 않는다
5. **모든 분석에는 반드시 데이터 근거가 있어야 함**——출처를 첨부
6. **인내심 있게 기다리기**——4개 에이전트의 리서치에는 몇 분이 걸리므로, 사용자에게 실시간으로 진행 상황을 업데이트
7. **반편향 의식**——team-lead는 종합할 때 반드시 다음을 평가해야 한다: 각 에이전트의 분석이 자료 풍부도에 의해 제약받지 않았는가? 시장 컨센서스와 과도하게 수렴하지 않았는가? 최종 보고서에는 "정보 풍부도 등급"과 "AI 리서치 한계 선언"이 포함되어야 함
8. **정보가 희소할 때의 정직 원칙**——보고서에 "데이터 부족"이라고 여백으로 남겨두는 편이, 추측으로 프레임워크를 채워 확실성을 위장하는 것보다 낫다
