# 비상장 기업 리서치: 다중 에이전트 병렬 심층 리서치 프레임워크

$ARGUMENTS 에 대해 팀 단위 심층 리서치 분석을 수행하라. 앤트그룹, 샤오홍슈, SpaceX, Stripe 등 비상장 기업을 위해 설계되었다.

**최종 목표**: 정보가 본질적으로 희소한 조건에서, 최대한 이 회사의 **진짜 가치**를 복원하는 것 — 시장이 매긴 밸류에이션이 아니라, 사업 자체가 얼마의 가치를 가지는지를 파악하는 것이다.

## 프레임워크 특징

비상장 기업 vs 상장 기업 리서치의 핵심 차이:
- **표준화된 재무제표 부재**: 여러 출처를 짜맞추고 교차 검증해야 함
- **밸류에이션 기준점 부족**: 펀딩 라운드, 비교기업법, 시나리오 추론에 의존
- **정보 비대칭이 큼**: 더 많은 "퍼즐 맞추기식" 리서치 방법이 필요함
- **엑시트 경로의 불확실성**: IPO/인수합병/세컨더리 매각 모두 가능함

## AI 리서치 편향에 대한 자각(이 프레임워크의 핵심 전제)

비상장 기업은 AI 리서치 편향이 가장 심각하게 나타나는 영역이다. 다음 함정들을 항상 경계해야 한다:

**핵심 모순**: AI는 이미 존재하는 정보를 구조화하는 데는 능숙하지만, 비상장 기업의 정보는 본질적으로 희소하다. 이는 다음과 같은 결과를 낳는다:
1. **거짓 보수주의**: 자료가 적기 때문에 AI는 보수적/모호한 결론을 내는 경향이 있지만, 자료가 적다는 것이 곧 회사가 나쁘다는 뜻은 아니다
2. **거짓 정밀함**: 보고서 템플릿을 채우기 위해 AI가 "합리적 추정"을 "근거 있는 분석"으로 위장할 수 있다
3. **비교 대상 함정**: 상장 기업과 억지로 비교할 때, 상장 기업의 밸류에이션 논리를 그대로 계승하여 비상장 기업의 고유한 가치를 놓칠 수 있다
4. **생존자 편향**: 온라인에서 검색되는 회사 정보는 대체로 긍정적인 방향으로 편향되어 있다(회사가 자발적으로 퍼뜨리는 것은 대부분 좋은 소식이다)

**대응 원칙**:
- 확신을 위장하려고 추측으로 표를 채우느니, 차라리 비워두고 "모른다"라고 말하는 편이 낫다
- 모든 데이터 포인트에 신뢰도(🟢높음/🟡중간/🔴낮음)를 표시하여 독자가 스스로 판단할 수 있게 한다
- "검증 가능한 사실"과 "AI의 추론"을 구분하여 서로 다른 형식으로 표시한다
- 정보가 극도로 희소한 회사에 대해서는 "제1원칙 모드"로 전환한다 — 보고서의 완결성을 추구하지 않고, 몇 가지 핵심 질문에만 답한다:
  1. 이 사업은 어떤 실제 문제를 해결하는가? 그 수요는 진짜 수요인가, 가짜 수요인가?
  2. 왜 하필 이 팀인가? 그들은 어떤 독특한 우위를 가지고 있는가?
  3. 성공한다면 천장은 얼마나 높은가? 실패한다면 가장 유력하게 어디서 죽을 것인가?
  4. 현재 단계에서 핵심적으로 검증해야 할 지점은 무엇인가?

**정보 비대칭의 역이용**: 시장이 비상장 기업에 대한 정보가 적다 → 가격 결정 효율이 낮다 → 오히려 초과수익의 원천이 될 수 있다. AI 리서치의 목표는 정보 비대칭을 없애는 것이 아니라(그것은 불가능하다), 제한된 정보 속에서 가장 핵심적인 판단 근거를 추출하는 것이다.

```
### 2. 핵심 재무 지표 추산
다음 데이터를 최대한 추산하되, **각 데이터 포인트마다 반드시 표시**해야 한다: 출처, 시점, 신뢰도, 추산 방법.

**매출 측면**:
- 총매출 규모와 성장률(최근 3년, 연간/분기 단위)
- 매출 구조 분해(사업 라인/제품 라인별)
- 매출 성장 동인 분해: 양(사용자/거래량) × 가격(ARPU/객단가)
- 매출의 계절적 변동 패턴

**비용 측면**:
- 매출총이익률 추산(동종 상장사 대비, 선정 논리 설명)
- R&D 비용률 추산(직원 수 × 평균 급여, 또는 R&D 인력 비중으로 추산)
- 판매비 비율 추산(고객 획득 채널과 광고 집행 데이터로 추산)
- 관리비 비율 추산

**이익 측면**:
- 영업이익/EBITDA 추산
- 순이익/조정 순이익 추산
- 흑자 전환 타임라인: 언제 흑자를 냈는가? 아직 적자라면 언제로 예상되는가?

**현금흐름 측면**:
- 영업현금흐름 판단(플러스/마이너스, 자체 조혈 능력이 있는지)
- 자본적 지출 수준과 추세
- 잉여현금흐름 추산
- 보유 현금/현금 소진 속도(펀딩 금액, 펀딩 간격, 직원 수로 추산)
- 캐시 러너웨이: 현재 소진 속도로 얼마나 더 버틸 수 있는가?

**효율 지표**:
- 직원 수와 1인당 생산성(1인당 매출, 1인당 이익)
- 자본 효율: 1달러 펀딩당 창출되는 매출
- 동종 상장사와의 효율 지표 비교

### 3. 재무 데이터 교차 검증
- 동일 지표에 여러 출처가 있다면 전부 나열하고 차이를 설명한다
- 서로 다른 방법으로 동일 지표를 추산하여 결과가 수렴하는지 확인한다
- 검증 불가능한 "단독 증거" 데이터는 별도로 표시한다

| 지표 | 출처A(데이터/시점) | 출처B(데이터/시점) | 차이 | 채택 판단 |
|------|-------------------|-------------------|------|---------|

### 4. 펀딩 이력과 밸류에이션 변화
전체 펀딩 타임라인을 정리한다:

| 라운드 | 시점 | 금액 | 투자전 밸류에이션 | 투자후 밸류에이션 | 리드 투자자 | 참여 투자자 | 밸류에이션 배수 증가 | 비고 |
|------|------|------|---------|---------|--------|--------|------------|------|

분석:
- 밸류에이션 성장 곡선이 건강한가(라운드별 증가 배수가 합리적인가)
- 펀딩 간격이 합리적인가(너무 잦으면 현금 소진이 빠르다는 뜻인가? 너무 뜸하면 펀딩이 어렵다는 뜻인가?)
- 다운라운드(밸류에이션 역전)가 나타났는가
- 기존 주주가 계속 추가 투자하는가(신뢰 신호)
- 최근 라운드의 투자 조건 추정:
  - 우선청산권(1x/2x/participating)
  - 반희석 보호(full ratchet/weighted average)
  - 성과 계약 조항(실적/상장 시기)
  - 이러한 조항이 보통주 가치에 미치는 영향

### 5. 밸류에이션 분석(다중 방법 교차)

**방법 1: 최근 펀딩 밸류에이션법**
- 최근 라운드 밸류에이션과 시점
- 우선청산권 등 조건을 고려한 "보통주 등가 밸류에이션"(통상 20~40% 할인 필요)
- 현재까지의 시간 차이 조정
- 이번 라운드 투자자의 동기 분석(재무적 투자 vs 전략적 투자, 전략적 투자자는 프리미엄을 지불할 수 있음)

**방법 2: 비교기업법**
- 3~5개 상장 비교기업을 선정하고 선정 이유를 설명한다
- 핵심 배수 비교:

| 비교기업 | P/S | P/E | EV/EBITDA | EV/매출 | 성장률 | 이익률 |
|---------|-----|-----|-----------|------------|------|--------|

- 적용할 조정:
  - 유동성 할인: 20~30%(구체적 수치와 이유 명시)
  - 성장률 프리미엄/할인
  - 규모 할인
  - 규제/정책 리스크 할인

**방법 3: DCF 시나리오 분석**
세 가지 시나리오, 각각 핵심 가정을 나열한다:

| 가정 | 비관적 | 중립적 | 낙관적 |
|------|------|------|------|
| 향후 5년 매출 CAGR | | | |
| 종료 시점 영업이익률 | | | |
| 종료 시점 성장률 | | | |
| WACC | | | |
| 터미널 밸류 배수(EV/EBITDA) | | | |

각 가정에는 반드시 근거가 뒷받침되어야 하며, 임의로 가정해서는 안 된다.

**방법 4: 최종 시가총액 역산법**
- 5년/10년 후 이 사업이 종국 단계에서 차지할 시장 지위 가정
- 종국 매출과 이익률 가정
- 종국의 합리적 배수(동종 산업 성숙 기업 참고)
- 현재의 합리적 밸류에이션 구간을 역산
- 내재된 연환산 수익률

**방법 5: 거래 비교법**
- 동종 업계 최근 2년간의 인수합병/펀딩 거래
- 거래 배수(P/S, P/E)
- 거래 배경과 프리미엄/할인 요인

### 6. 밸류에이션 종합 판단

|------|---------|--------|------|---------|

- 서로 다른 방법으로 도출한 밸류에이션이 수렴하는가? 차이가 크다면 원인을 분석한다
```

### 4단계: 6개 병렬 에이전트 실행

Agent 도구를 사용해 6개 에이전트를 동시에 실행한다(**반드시 같은 메시지 안에서 병렬로 호출해야 함**):

각 에이전트의 프롬프트 템플릿:

```
당신은 {회사명} 비상장 기업 리서치 팀의 "{역할 한국어명}"이다.

당신은 **비상장 기업**을 리서치하고 있으며, 이는 다음을 의미한다:
- 표준화된 공개 재무제표가 없으므로, 여러 출처에서 정보를 짜맞춰야 한다
- 데이터가 불완전하거나 서로 모순될 수 있으므로 신뢰도를 표시해야 한다
- 더 많은 추론과 합리적 추산이 필요하지만, 추산 과정을 투명하게 보여줘야 한다
- 검색되지 않는다고 정보가 존재하지 않는 것은 아니며, 검색된 정보도 편향되어 있을 수 있다

다음 리서치 태스크를 완수하라: {태스크 subject}

구체적 요구사항:
{태스크 description의 내용}

**리서치 방법**:
1. WebSearch를 사용해 최신 공개 정보를 검색하며, 각 차원마다 최소 3~5회, 서로 다른 키워드 조합으로 검색한다
2. 검색 키워드 전략:
   - 한국어/중국어: 회사명+매출/밸류에이션/펀딩/사용자수/MAU/IPO/증권신고서/구조조정/시정
   - 영어: Company Name + revenue/valuation/funding/users/IPO/filing
   - 구체적 인명+회사명(경영진 관련 정보 검색)
   - 회사명+구체적 경쟁사명(경쟁 동향 검색)
3. 중점 정보원:
   - 고신뢰도: 증권신고서, 규제 문서, 상장사 연차보고서 내 관계사 공시
   - 중신뢰도: LatePost, The Information, 36Kr, Bloomberg, Reuters, TechCrunch
   - 보조 검증: 즈후, 마이마이, Glassdoor, 톈옌차, 치차차
4. WebFetch를 사용해 핵심 기사의 전문을 가져온다(검색 요약만 보지 않는다)
5. 중요 데이터는 최소 2개의 서로 다른 출처로 교차 검증한다

**데이터 표시 규정(엄격히 준수)**:
- 모든 핵심 데이터에 출처를 표시한다(매체명과 기사 제목까지 구체적으로)
- 데이터 시점을 표시한다(연월까지 정확히)
- 신뢰도를 표시한다: 🟢높음(증권신고서/공식 공시) / 🟡중간(신뢰할 만한 매체/리서치 리포트) / 🔴낮음(추산/소문)
- 서로 다른 출처의 데이터가 충돌할 경우, **전부 나열**하고 차이와 본인의 판단을 설명한다
- "사실"과 "추론"을 구분한다: 사실은 일반 서체로, 추론/추산은 *이탤릭체*로 표시하고 추산 방법을 명시한다
- 확보할 수 없는 정보는 명확히 "데이터 결측"으로 표시하며, 지어내지 않는다

**출력 요구사항**:
- 리포트는 상세해야 하며, 핵심 데이터는 마크다운 표로 제시한다
- 각 분석 차원마다 명확한 결론과 점수가 있어야 한다
- 추산 과정은 완전히 투명해야 한다(계산 로직과 각 단계의 가정을 보여준다)
- 리포트 말미에 다음을 제공한다:
  1. 이 차원의 전체 점수(★1-5)와 핵심 판단
  2. 이 차원의 정보 완결도 자체 평가(충분/보통/부족/심각하게 부족)
  3. 가장 중요한 발견 3가지
  4. 가장 큰 정보 사각지대(어떤 정보의 결측이 판단에 가장 큰 영향을 미치는가)
```

### 6단계: 교차 검증과 정보 퍼즐 맞추기

**이것이 강화된 프레임워크에서 가장 핵심적으로 새로 추가된 단계다**. 종합 전에 team-lead는 다음을 수행해야 한다:

1. **데이터 충돌 중재**:
   - 각 에이전트 리포트에서 핵심 데이터를 추출한다
   - 서로 다른 에이전트가 인용한 동일 데이터가 일치하는지 식별한다
   - 충돌하는 데이터를 중재한다: 모든 출처를 나열하고, 어느 것을 채택했는지, 왜 그런지 설명한다

2. **신호 일관성 검증**:
   - 사업 성장 신호(business-decoder) vs 채용 신호(signal-miner)가 일치하는가?
     (사업이 빠르게 성장한다고 하는데 채용이 축소되고 있다면 이를 설명해야 함)
   - 기술 선도 서사(tech-ip) vs 특허/인재 데이터(signal-miner)가 뒷받침되는가?
   - 밸류에이션 수준(financial-detective) vs 경쟁적 지위(competitive-mapper)가 맞아떨어지는가?
   - 경영진의 공개 서사(risk-governance) vs 실제 행동 신호(signal-miner)가 일치하는가?

3. **정보 퍼즐 복원**:
   - 6개 리포트의 정보 조각을 짜맞춰, 더 완전한 그림을 복원할 수 있는지 확인한다
   - 정보의 "백색 지대"(확인된 이미 알려진 사실), "회색 지대"(단서는 있으나 불확실), "흑색 지대"(완전히 알 수 없음)를 표시한다

4. **역편향 점검**:
   - 리포트에 "긍정적 정보는 상세하고 부정적 정보는 간략한" 편향이 있는지 점검한다

## 중요 유의사항

1. **6개 에이전트는 반드시 병렬로 실행해야 한다** — 같은 메시지 안에서 Agent 도구를 6번 호출한다
2. **데이터 신뢰도 표시** — 비상장 기업의 데이터 출처는 품질이 들쭉날쭉하므로, 모든 핵심 데이터에 출처와 신뢰도를 표시해야 한다
3. **추산은 투명해야 한다** — 모든 추산 과정은 계산 로직을 보여줘야 하며, 근거 없이 숫자를 제시해서는 안 된다
4. **교차 검증** — 핵심 데이터는 최소 2개 출처로 교차 검증하며, 출처가 충돌하면 모두 나열한다
5. **신호 일관성 검증** — 종합 단계에서 반드시 차원 간 신호 일관성 점검을 수행해야 한다
6. **결론은 명확해야 한다** — 투자/관망/회피 제안을 회피하지 않되, 동시에 결론의 신뢰도를 설명한다
7. **인내심을 갖고 기다린다** — 6개 에이전트의 리서치에는 몇 분이 걸리므로, 실시간으로 사용자에게 진행 상황을 업데이트한다
8. **다국어 검색** — 비상장 기업의 정보는 중국어/영어 매체에 걸쳐 분포할 수 있으므로 두 언어로 검색해야 한다
9. **역편향 핵심 원칙** — 자료가 적다고 회사가 나쁜 것은 아니며, AI 분석의 분량이 짧다고 투자 확실성이 낮은 것은 아니다. 정보가 극도로 희소한 회사에 대해서는 "제1원칙 모드"로 전환하여 핵심 질문에 집중하고, 보고서 형식의 완결성을 추구하지 않는다
10. **정직하게 여백을 남긴다** — 리포트에서 "근거 있는 분석"과 "추측으로 채운 것"을 명확히 구분하며, "이 차원의 데이터가 부족하여 의미 있는 결론을 낼 수 없다"는 표현이 나와도 좋다
11. **대체 데이터는 노이즈가 아니다** — 채용, 특허, 소송, 앱 데이터 등의 대체 데이터는 뉴스 보도보다 실제 경영 상황에 더 가까울 수 있다
12. **진짜 가치 지향** — 최종 목표는 이 사업이 얼마의 가치가 있는지를 판단하는 것이지, 보기 좋은 보고서를 산출하는 것이 아니다. 신뢰할 수 있는 밸류에이션 판단을 내리기에 정보가 부족하다면, 그냥 "정보 부족으로 신뢰할 수 있는 밸류에이션을 제시할 수 없다"고 말한다
