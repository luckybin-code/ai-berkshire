# 공급망 병목 헌터: AI 기반 글로벌 산업망 병목 차익거래

$ARGUMENTS 슈퍼트렌드에 대해 공급망 병목 스캔과 차익거래 기회 발굴을 수행한다.

## 핵심 철학

"AI가 어떤 종목을 추천하는가"를 묻지 않고, "이 트렌드가 계속 확장된다면, 어느 구간이 가장 먼저 부족해질 것인가"를 묻는다.

전통적인 투자 리서치는 선두 기업과 이미 알려진 트랙에만 주목한다. 이 시스템은 반대로 접근한다. **물리적 공급망의 목구멍(咽喉) 위치에서 출발해, 아무도 주목하지 않지만 공급이 끊기면 업계 전체가 멈춰서야 하는 회사를 찾는다.**

초과수익의 원천: 1차 병목(GPU, HBM, 전력)은 이미 충분히 가격에 반영되어 있다. 진짜 알파는 **2차, 3차 구간**에 있다—광모듈, 레이저, InP 기판, SOI 웨이퍼, 에피택시 장비, 웨이퍼 레벨 테스트, IC 캐리어 기판, 특수 유리섬유 등.

## 1단계: 슈퍼트렌드 확인

### 1.1 트렌드 선별 기준

작은 테마에서 환상을 찾지 않고, 아래 조건을 모두 충족하는 슈퍼트렌드만 추적한다.

| 기준 | 요구사항 | 검증 방법 |
|------|------|---------|
| 지속성 | 최소 3-5년의 확실한 성장 | 업계 전망, 자본지출 계획 검색 |
| 물리성 | 실제 하드웨어/소재/설비 구축이 필요함 | "소프트웨어 업그레이드"와 "물리적 증설"을 구분 |
| 규모성 | 글로벌 자본지출 > 연 500억 달러 | 주요 기업의 capex 가이던스 검색 |
| 가속성 | 수요 증가 속도 > 공급 증설 속도 | 수요 증가율 vs 생산능력 확장 계획 비교 |

### 1.2 현재 추적 중인 슈퍼트렌드 목록

매번 실행 시 갱신하며, 초기 목록은 다음과 같다.

1. **AI 인프라 구축** — 데이터센터, GPU 클러스터, 네트워크 상호연결, 전력
2. **에너지 전환** — 원전 재가동, 전력망 업그레이드, 에너지 저장
3. **국방 현대화** — 서방 국방비 상승 사이클, 공급망 재편
4. **반도체 재산업화** — 미·유럽·일본의 보조금 공장 건설, 설비/소재 병목
5. **우주 경제** — 위성 인터넷, 발사 빈도 급증

사용자가 구체적인 트렌드(예: "AI 인프라")를 지정한 경우 해당 트렌드에만 집중한다.

### 1.3 트렌드 검증 출력

```
트렌드 이름:
핵심 동력: (한 문장)
이미 발생한 검증 사건(최소 3건):
  1. [날짜] [사건] [출처]
  2.
  3.
자본지출 규모: 글로벌 약 $XX억/년, 증가율 YY%
수급 갭 판단: 수요 증가율 > 공급 증설 속도? 예/아니오/불확실
트렌드 확인: ✅ 추적 가능 / ❌ 근거 부족, 추적 보류
```

## 2단계: 공급망 물리적 분해

### 2.1 계층별 분해 프레임

**개념 수준에 머물지 말고 물리적 실체까지 분해한다.**

```
Layer 0(최종재): 최종 제품/서비스
    │
Layer 1(핵심 부품): 시장에서 이미 충분히 주목받는 핵심 하드웨어
    │                 ⬆ 가격 반영 충분, 알파 제한적
    │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
    │                 ⬇ 주목도 낮음, 알파 집중 구간
    │
Layer 2(서브 부품/소재): 핵심 부품을 뒷받침하는 부품과 소재
    │
Layer 3(상류 설비/원재료): 서브 부품 제조에 필요한 설비와 원재료
    │
Layer 4(인프라): 전력, 냉각, 부지, 인력, 인증
```

### 2.2 AI 인프라를 예로 든 분해 템플릿

```
Layer 0: AI 모델 학습/추론 서비스
Layer 1: GPU/가속기, HBM 메모리, 서버, 데이터센터
Layer 2(중점 스캔 구간):
  ├─ 네트워크 상호연결: 광모듈, 광섬유, 스위칭 칩, 구리 케이블
  ├─ 광통신 핵심: 레이저(EML/VCSEL/CW), 변조기, 광검출기
  ├─ 반도체 소재: InP 기판, GaAs 기판, SOI 웨이퍼, SiC 기판
  ├─ 첨단 패키징: CoWoS 캐리어 기판, HBM TSV, ABF 기판 필름
  ├─ PCB/캐리어 기판: 고주파 고속 PCB, IC 캐리어 기판, 특수 유리섬유포
  ├─ 테스트: 웨이퍼 레벨 테스트(프로브 카드), 번인 테스트, ATE
  ├─ 방열/냉각: 액체 냉각 시스템, CDU, 침지식 냉각액
  └─ 전력 연결: 버스바, UPS, 배전반, 변압기
Layer 3:
  ├─ 에피택시 설비: MOCVD, MBE
  ├─ 포토리소그래피/에칭: 특수 파장 리소그래피, InP 에칭
  ├─ 원재료: 고순도 금속(인듐, 갈륨, 게르마늄), 특수가스, 타겟재
  └─ 인증/표준: MSA 표준, Telcordia 인증
Layer 4:
  ├─ 전력: 원자력, 천연가스 발전, 송배전
  ├─ 냉각수/방열 인프라
  └─ 데이터센터 부지/인허가
```

## 3단계: 병목 식별 — "목구멍" 찾기

### 3.1 병목 판정 6대 기준

Layer 2-3의 각 구간에 대해 항목별로 평가한다.

| # | 기준 | 질문 | 점수 |
|---|------|------|------|
| 1 | **공급 집중도** | 글로벌 공급업체 ≤3곳인가? | 🔴 ≤2곳 / 🟡 3-5곳 / 🟢 >5곳 |
| 2 | **증설 주기** | 신규 생산능력 확보에 얼마나 걸리는가? | 🔴 >2년 / 🟡 1-2년 / 🟢 <1년 |
| 3 | **대체 난이도** | 다른 기술/소재로 대체 가능한가? | 🔴 대체 불가 / 🟡 부분 대체 가능 / 🟢 대체 용이 |
| 4 | **가동률** | 현재 생산능력 가동률은? | 🔴 >90% / 🟡 70-90% / 🟢 <70% |
| 5 | **수요 증가율** | 하류 수요 증가율은? | 🔴 >연 50% / 🟡 20-50% / 🟢 <20% |
| 6 | **고객 검증 주기** | 신규 공급업체 진입 시 검증에 얼마나 걸리는가? | 🔴 >1년 / 🟡 6-12개월 / 🟢 <6개월 |

**병목 등급**:
- 🔴🔴🔴 ≥4개 → **S급 병목** (단일 장애점 수준, 최우선순위)
- 🔴🔴 3개 → **A급 병목** (심각하게 제약됨)
- 🔴 1-2개 → **B급 병목** (부담은 있으나 관리 가능)
- 🔴 없음 → 병목 아님, 건너뜀

### 3.2 병목 지도 출력

```
공급망 병목 지도 — {트렌드 이름}
갱신일: YYYY-MM-DD

S급 병목(단일 장애점):
  1. [구간명] — [이유 한 줄] — 공급업체: [회사 목록]
  2.

A급 병목(심각하게 제약됨):
  1.
  2.

B급 병목(부담 있음):
  1.
  2.

최근 변화(지난 스캔 대비):
  - [신규/상향/하향/해제] [구간명] — [이유]
```

## 4단계: 기업 선별 — 병목에서 종목으로

### 4.1 각 S급, A급 병목에 대해 관련 상장기업을 모두 찾는다

검색 방식:
- WebSearch "{병목 구간} supplier listed company"
- WebSearch "{병목 구간} manufacturer stock"
- WebSearch "{병목 제품} market share company"

### 4.2 1차 선별 기준(빠른 필터링)

| 기준 | 요구사항 | 이유 |
|------|------|------|
| 상장 여부 | 이미 상장(A주/홍콩/미국/일본/대만/유럽) | 거래 가능해야 함 |
| 병목 사업 비중 | 병목 구간 매출 비중 >30% | 순도(纯正度) |
| 시가총액 | 100억 달러 미만 우선 | 대형주는 이미 충분히 가격 반영됨 |
| 유동성 | 일평균 거래대금 >100만 달러 | 매매 가능해야 함 |

### 4.2.1 밸류에이션 체크(반드시 수행, 생략 불가)

**병목이 진짜라는 것 ≠ 투자 기회.** 모든 기업에 대해 PS, PE를 계산하여 보고서에 반드시 표기해야 한다. 아래 복합 조건으로 밸류에이션 과열 여부를 판단한다.

#### 밸류에이션 적신호 (아래 중 하나라도 해당 → 신호 강도 상한 ★★, "⚠️ 밸류에이션 과열" 표기)

1. **시가총액 > TAM의 20%**: 회사 시가총액이 접근 가능한 시장 규모의 20%를 이미 초과 — 성장 기대가 과도하게 선반영되었음을 의미
2. **PS > 30배 且 매출 증가율 < 100%**: 고밸류에이션인데 성장률이 이를 뒷받침하지 못함. 증가율 >100%인 기업은 PS 레드라인에서 면제되나, 여전히 "⚠️ 고밸류에이션은 지속적인 고성장 검증 필요" 표기
3. **시가총액 > 5년 낙관적 매출 전망의 10배**: 가장 낙관적인 가정이 전부 실현되더라도 현재 가격은 여전히 과도함
4. **증자 후 60일 이내 주가 2배 상승**: 심리 주도 특징이 뚜렷함, 신호 강도 한 단계 하향

#### 밸류에이션 황신호 (추가 설명 필요, 없으면 등급 하향)

1. **적자 + PS > 15배**: ★★★까지는 허용되나 반드시 수익성 확보 경로와 시기를 설명해야 함
2. **PS가 동종 흑자 기업의 5배 이상**: 반드시 프리미엄의 근거를 설명(시장점유율, 증가율 차이, 진입장벽 차이)
3. **PE > 80배**: PEG를 계산하고 증가율이 이를 뒷받침하는지 설명 필요

#### 밸류에이션 청신호 (가점 요인)

- PS < 10배 且 매출 성장 중 → 신호 강도 한 단계 상향 가능
- PE < 30배 且 해자 보유 → "밸류에이션이 안전마진을 갖춤"으로 표기

#### 밸류에이션 합리성 검증(필수)

각 종목에 대해 답한다: "현재 시가총액으로 매수하여, 가장 낙관적인 시나리오가 전부 실현되고 10년 후 25배 PE로 엑시트한다면, 연환산 수익률은 얼마인가?" 연환산 수익률 <10% → "현재 가격은 안전마진을 갖추지 못함" 표기.

**주의**: 밸류에이션 체크의 목적은 "PS 100배의 적자 기업"과 같은 명백한 오류를 추천하는 것을 방지하기 위함이지, 모든 고밸류에이션 초기 기업을 배제하기 위함이 아니다. 핵심은 증가율, TAM, 경쟁 구도가 현재 밸류에이션을 뒷받침할 수 있는가이며, 일률적인 판단이 아니라 구체적인 분석이 필요하다.

### 4.3 심층 선별 항목

1차 선별을 통과한 기업에 대해 개별적으로 평가한다.

```
## {회사명}({티커})

**병목 포지션**:
- 공급망 내 구체적 위치
- 시장점유율: 글로벌 X위, 비중 XX%
- 고객 목록(알려진 경우)

**생산능력과 증설**:
- 현재 생산능력 / 가동률
- 증설 계획 / 일정
- 증설에 필요한 자금 vs 보유 현금

**재무 스냅샷**:
- 시가총액 / 매출 / 이익 / 증가율
- 병목 사업 매출 비중
- 매출총이익률 추세(병목이 심할수록 매출총이익률은 상승해야 함)

**리스크 체크리스트**:
- [ ] 대체 기술 리스크: 우회 가능한가?
- [ ] 희석 리스크: 대규모 증자/전환사채가 있는가?
- [ ] 지정학적 리스크: 민감 지역에 있거나 수출 규제 대상인가?
- [ ] 경영진 리스크: 불량 이력이 있는가?
- [ ] 고객 집중 리스크: 단일 고객에 과도하게 의존하는가?
- [ ] 밸류에이션 과열: 현재 밸류에이션이 이미 3년치 성장을 반영했는가?

**병목 지속성 판단**:
- 이 병목은 언제 해제될 것인가?
- 해제 후 이 회사에는 무엇이 남는가?
- 일회성인가 지속적인가?
```

## 5단계: 교차 검증 — 하나의 스토리만 듣지 않는다

### 5.1 정방향 검증

| 검증 항목 | 질문 | 검색 방식 |
|--------|------|---------|
| 고객 검증 | 주요 고객이 이미 계약/도입했는가? | 회사 공시, 고객사 실적 발표 언급 검색 |
| 매출 검증 | 병목이 이미 매출 성장에 반영되었는가? | 최근 2-3개 분기 실적 검색 |
| 가격 검증 | 제품 가격이 인상되고 있는가? | 업계 견적, 애널리스트 리포트 검색 |
| 생산능력 검증 | 생산능력이 실제로 타이트한가? | 납기 데이터, 고객 불만 검색 |
| 자본 검증 | 증설 자본지출이 있는가? | 회사 capex 가이던스 검색 |

### 5.2 역방향 검증(멍거식 부정)

| 역방향 질문 | 의미 |
|---------|------|
| 왜 똑똑한 사람들은 이 주식을 사지 않는가? | 이미 알려진 약세 논거 찾기 |
| 이 병목은 우회할 수 있는가? 대체 경로는 무엇인가? | 기술 경로 리스크 |
| 중국/다른 경쟁자가 생산능력을 빠르게 복제할 수 있는가? | 공급 충격 리스크 |
| 최종 수요가 50% 둔화된다면 이 회사는 어떻게 되는가? | 하방 민감도 |
| 경영진이 과거 고점에서 증자로 희석시킨 적이 있는가? | 경영진 신뢰도 |
| 현재 밸류에이션은 어떤 성장 가정을 내포하는가? | 밸류에이션 합리성 |

### 5.3 신호 교차 검증

- 동일 병목의 여러 기업이 모두 상승하고 있는가? (업계 검증)
- 하류 고객이 실적 발표에서 공급 부족을 언급했는가? (고객 검증)
- 업계 협회/연구기관에 관련 데이터가 있는가? (제3자 검증)

### 6.1 병목 기회 순위표

|------|------|------|------|--------|-----|-----|---------|---------|---------|---------|---------|---------|
| 1 | | | | | x | x | | S/A | | | ★1-5 | 합리/다소 높음/과열 |

**필수 항목**: 시가총액, 연매출, PS, PE는 필수 기재 항목이며 "미확인"으로 생략할 수 없다. 재무 데이터를 확보하지 못한 경우 신호 강도는 ★★를 초과할 수 없다.

신호 강도 평가(밸류에이션 체크 결과가 등급에 직접 영향):
- ★★★★★ 다중 교차 검증, 고객 도입 완료, 매출에 이미 반영, 밸류에이션 청신호(PS 합리적이고 흑자이거나 흑자 근접)
- ★★★★ 대부분 검증 통과, 밸류에이션 청신호 또는 황신호(설명 첨부 필요)
- ★★★ 논리는 성립하나 일부 미검증, 밸류에이션 황신호 수용 가능(예: 고성장 초기 기업)
- ★★ 초기 신호이거나 병목 논리는 성립하나 밸류에이션 적신호(시가총액>TAM 20%, PS>30배且증가율 부족, 시가총액이 5년 전망을 훨씬 초과 등)
- ★ 순수 개념, 미검증

## 7단계: 축적 갱신 — 병목 지도의 동적 유지관리

### 7.1 매 실행 시 증분 갱신

1. 이미 식별된 병목이 여전히 유효한지 확인
   - 신규 공급업체가 진입했는가?
   - 생산능력이 이미 병목을 해소할 만큼 확장되었는가?
   - 대체 기술에 돌파구가 있는가?

2. 새로 등장한 병목 스캔
   - 최근 7일간의 supply chain / shortage / bottleneck 뉴스 검색
   - 실적 시즌 중 공급망 관련 공시 확인

3. 병목 등급 갱신(상향/하향/해제)

### 7.2 상태 파일

`reports/bottleneck-map/` 디렉토리에서 다음을 유지관리한다.
- `master-map.md` — 병목 총지도(지속 갱신)
- `watchlist.md` — 관찰 목록(지속 갱신)
- `YYYY-MM-DD/` — 하루 한 폴더, 해당 날짜의 모든 스캔 리포트 포함
- `deep-dive/` — 심층 분석한 기업은 별도 파일로 작성

## 시간별 스캔 모드(정기 작업용)

매시간 1회 실행하며, "건질 게 있을 때만 리포트를 낸다"는 방식을 채택한다.

### 스캔 절차(매시간)

1. **뉴스 스캔**: 지난 1-2시간의 공급망 관련 뉴스 검색
   - 키워드: supply chain bottleneck, shortage, capacity constraint, allocation, lead time, sole source, 병목, 품절, 생산능력, 가격인상
   - 커버리지: 영문+중문 소스
2. **시장 신호**: 이미 추적 중인 기업의 주가 변동 확인(특히 이상 변동 >5%에 주목)
3. **실적/공시**: 병목 관련 기업이 실적을 발표하거나 중대 공시를 냈는지 확인
4. **밸류에이션 기회**: 관찰 목록의 기업이 전반적인 시장 하락 등으로 매수 구간에 진입했는지 확인
5. **리포트 발행 여부 판단**:
   - 새로운 병목 신호, 명확한 종목 기회, 중대한 상태 변화 → **리포트 발행**
   - 새로운 발견 없음 → **리포트 미발행**, 로그에만 "이번 회차 신호 없음" 기록

### 리포트 출력 규칙

**하루에 폴더 하나**: `reports/bottleneck-map/YYYY-MM-DD/`

**파일 명명 규칙**(파일명만 보고 종목 유무를 한눈에 알 수 있도록):

| 상황 | 파일명 형식 | 예시 |
|------|-----------|------|
| 명확한 종목 발견 | `HH-MM-종목코드1-종목코드2.md` | `09-00-FORM-IBDN.md` |
| 병목 신호는 있으나 명확한 종목 없음 | `HH-MM-신호스캔.md` | `14-00-신호스캔.md` |
| 새로운 발견 없음 | 파일 생성 안 함 | — |

**파일명 속 종목 코드 = 밸류에이션 체크를 통과해 심층 연구할 가치가 있는 기업.** 신호 스캔 단계에만 등장하고 밸류에이션이 미달인 기업은 파일명에 넣지 않는다.

## AI 리서치 편향 자각

| 편향 | 표현 | 대응 |
|------|-----|------|
| 선두주 선호 | 검색 결과가 대형주에 편중됨 | 소형 공급업체를 의도적으로 검색, "small cap" 키워드 추가 |
| 영어 선호 | 일/한/대만 기업 누락 | 반드시 일본/한국/대만 시장의 공급업체 검색 |
| 내러티브 선호 | "AI 개념" 라벨에 끌림 | 실제 공급망 위치만 보고 시장 라벨은 보지 않음 |

| 시효 편향 | 오래된 정보에 의존 | 최근 30일 데이터를 우선 검색 |

## 핵심 원칙(최우선순위)

1. **AI가 종목을 추천하게 하지 말고, AI가 공급망을 분해하게 하라** — 답보다 질문이 중요하다
2. **물리 우선** — 실제 물리적 제품/소재/설비가 필요한 구간에만 주목한다
3. **2차, 3차 구간** — 이미 충분히 가격에 반영된 선두주는 쫓지 않는다
4. **교차 검증** — 모든 결론에는 최소 2개의 독립적인 정보원이 있어야 한다
5. **불확실성에 정직하라** — 데이터를 찾지 못하면 "데이터 부족"이라 쓰고, 추측으로 채우지 않는다
6. **병목에는 시효가 있다** — 모든 병목은 결국 해제되며, 핵심은 시간 창(time window)을 판단하는 것이다
7. **소형주 ≠ 좋은 기회** — 소형주도 부실 기업일 수 있으므로 반드시 재무 품질 검증을 거쳐야 한다
8. **병목이 진짜라는 것 ≠ 투자 기회** — 어떤 회사가 가장 타이트한 병목에 앉아 있더라도, PS>30배이거나 여전히 적자라면 현재 가격은 매수 시점이 아니다. **밸류에이션은 넘을 수 없는 문턱이며, 병목의 순도, 신호 강도, 내러티브의 매력에 가려져서는 안 된다.** 이미 오른 병목주를 놓치는 한이 있어도, PS 100배에 적자 기업을 사서는 안 된다
9. **CLAUDE.md의 객관성 원칙을 준수한다** — 강세를 전제하지 않고, 데이터가 먼저, 결론은 나중에
