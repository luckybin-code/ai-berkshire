# 투자 리서치: 버핏-멍거-돤융핑-리루 4대 거장 종합 분석 프레임워크

$ARGUMENTS에 대해 체계적인 투자 리서치 분석을 수행한다.

## 리서치 프레임워크

버핏, 멍거, 돤융핑, 리루 4대 투자 거장의 방법론을 기반으로, 다음 7개 모듈 순서로 리서치를 수행한다:

### 사전 단계: AI 리서치 편향 자각 (필수 실행)

리서치를 시작하기 전, 먼저 해당 기업의 "AI 리서치 가능성"을 평가하고 잠재적 데이터 편향을 식별한다:

**정보 풍부도 등급**:
| 등급 | 특징 | AI 리서치 함정 | 대응 전략 |
|------|------|-----------|---------|
| A등급 (정보 풍부) | 상장 다년, 증권사 커버리지 많음, 언론 보도 밀도 높음 | 합의가 지나치게 강해, AI 출력이 시장 가격에 수렴하며 알파가 제한적 | 반대 검증에 집중: 똑똑한 사람들은 왜 사지 않는가? 간과된 리스크는 무엇인가? |
| B등급 (정보 보통) | 상장 1-3년, 커버리지 제한적, 일부 데이터는 추산 필요 | AI가 "합리적 추정"으로 공백을 메울 수 있어, 완전해 보이지만 실은 허위 확실성일 수 있음 | 추산 데이터마다 신뢰도를 표기하고, "근거 있는 추산"과 "근거 없는 채움"을 구분 |
| C등급 (정보 희소) | 갓 상장/비인기 종목/신흥 시장, 커버리지 거의 없음 | AI가 자료 부족으로 과도하게 보수적이 되어, "안 보임=안 좋음"으로 오판할 수 있음 | 제1원리 질문(아래 참고)을 사용해, 제한된 정보에서 비즈니스 본질을 추출 |

**C등급 기업의 제1원리 리서치법**:
공개 자료가 부족할 때는 "완전해 보이는" 보고서를 억지로 짜맞추려 하지 말고, 다음 근본 질문에 집중한다:
1. 고객은 누구인가? 왜 돈을 지불하는가? 대체 선택지가 있는가?
2. 재구매는 무엇에 의해 이루어지는가? 습관인가, 락인인가, 아니면 지속적으로 새로운 가치를 창출하기 때문인가?
3. 경쟁사가 100억을 가지고 이 사업을 복제할 수 있는가?
4. 경영진은 어떤 핵심 의사결정을 내렸는가? 이 결정들은 어떤 판단력과 가치관을 반영하는가?

**편향 자체 점검 목록** (리서치 전 과정에서 경계 유지):
- [ ] 나의 "확실성" 느낌은 사업 본질에서 오는가, 아니면 자료의 양에서 오는가?
- [ ] 이 회사의 자료량을 절반으로 줄이면 내 결론이 바뀌는가?
- [ ] AI 출력 분석이 시장 합의와 매우 유사한가? 그렇다면 나의 정보 우위는 어디에 있는가?
- [ ] "공개 자료는 적지만 사업 본질은 매우 좋을" 가능성이 저평가되고 있지는 않은가?

정보 풍부도 등급 결과를 보고서 서두에 기재하고, 최종 결론에서 "AI 리서치 신뢰도"와 "실제 투자 확실성"의 차이를 명시한다.

### 1단계: 데이터 수집

> **데이터 출처 규정**: `skills/financial-data.md` 참고. 모든 재무 데이터는 반드시 두 개의 독립된 출처에서 확보해야 하며, 오차가 1%를 초과하면 표시한다.
> - 미국주: macrotrends(주) + stockanalysis(부)
> - 홍콩주: aastocks(주) + macrotrends ADR(부)
> - A주: 동방재부(주) + 거차오자쉰(부)

Task 도구를 사용해 백그라운드 Agent를 실행하여, 웹에서 다음 데이터를 수집한다:

1. 매출 구조: 최근 회계연도 및 최근 4개 분기의 부문별 매출, 성장률, 매출총이익률
2. 재무 지표: 최근 5년 매출, 순이익, 매출총이익률, 영업이익률, 잉여현금흐름, 현금 보유량
3. 경쟁 구도: 시장 점유율, 주요 경쟁사 비교
4. 비즈니스 모델과 해자: 핵심 경쟁우위의 원천
5. 기술 역량: 핵심 기술 스택, R&D 투자
6. 경영진: 창업자/CEO 이력, 지분 비율, 핵심 의사결정 기록
7. 산업 전망: TAM(총주소가능시장), 성장 전망
8. 리스크 요인: 지정학, 규제, 공급망 등
9. 현재 밸류에이션: 시가총액, PE, PS, PEG, EV/Revenue
10. 매도/매수 양측의 핵심 논거

#### 데이터 교차 검증 (필수 실행, 금융 정밀 도구 사용)

데이터 수집 완료 후, **반드시 `tools/financial_rigor.py`를 호출해 핵심 데이터를 프로그램적으로 검증**해야 하며, LLM의 암산 오류를 방지해야 한다.

**반드시 검증해야 할 데이터 항목**:
- 총발행주식수 (거래소, Yahoo Finance, StockAnalysis 등 최소 2개 출처에서 확인)
- 현재 주가와 시가총액 (**주가×총발행주식수를 수동 계산해 보고서 시가총액과 대조, 단위 오류 방지**)
- 최근 회계연도 매출과 순이익 (회사 연차보고서+최소 1개 제3자 출처에서 확인)
- 현금 보유량과 순현금 (현금+단기투자-총부채, 기준 차이에 유의)
- 경영진 지분 비율 (경제적 지분과 의결권 구분, AB주 구조 유의)

**강제 검증 절차 (Bash로 도구 호출)**:

1단계 — 시가총액 검산 (정밀 십진수, 부동소수점 아님):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-market-cap \
  --price {주가} --shares {총발행주식수} --reported {보고된시가총액} --currency {통화}
```

2단계 — 핵심 데이터 다중 출처 교차 검증:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py cross-validate \
  --field {필드명} --values '{"출처1": 수치, "출처2": 수치}' --unit {단위}
```
매출, 순이익, 현금 보유량에 대해 각각 실행한다.

3단계 — 밸류에이션 지표 정밀 검산 (PE/PB/ROE/FCF Yield 등):
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py verify-valuation \
  --price {주가} --eps {EPS} --bvps {주당순자산} --fcf-per-share {주당FCF} --dividend {주당배당}
```

**검증 규칙**:
1. 각 핵심 데이터 항목은 최소 2개의 독립 출처를 확보한다
2. 출처 간 차이가 발견되면 회사 연차보고서/거래소 데이터를 우선 채택하고, 차이 원인을 명시한다
3. **계산이 포함된 모든 데이터는 반드시 도구로 검산해야 하며, LLM 암산은 금지한다**
4. 도구 출력 결과는 보고서 부록 "핵심 데이터 교차 검증 기록"에 직접 삽입한다
5. 도구가 ❌ 편차 과다를 보고하면, 원인을 규명한 후에만 분석을 계속할 수 있다

**흔한 오류 방지**:
- 시가총액 단위: 홍콩달러 억 vs 위안 억 vs 달러 억, 0 하나를 빠뜨리거나 더 붙이기 쉬움
- FCF 기준: 출처마다 자본지출의 정의가 다를 수 있음(리스, 인수 포함 여부)
- 부채 기준: 운용리스 부채 포함 여부
- 지분 비율: AB주 기업의 경제적 지분 ≠ 의결권

### 2단계: 비즈니스 본질 분석 — 돤융핑의 "좋은 사업"

분석 포인트:
- 이 사업의 본질을 한 문장으로 정의
- 매출 구조 분해(도표)
- 5년 수익성 추세(도표)
- 비즈니스 모델 캔버스: 일회성 판매 vs 구독/재구매? 하드웨어 vs 소프트웨어 vs 플랫폼?
- 생태계 락인/고객 고착 강도
- 매출총이익률 수준과 동종업계 비교, 왜 높은지/낮은지 설명
- 영업 레버리지 분석
- **돤융핑식 질문**: 이 사업의 좋은 점은 어디에 있는가? 한 문장으로만 설명한다면 무엇인가?

### 3단계: 해자 평가 — 버핏의 "경제적 해자"

5대 해자 유형을 하나씩 검증한다:

| 해자 유형 | 검증 방법 |
|-----------|---------|
| 브랜드/가격결정력 | 판매량 손실 없이 가격을 인상할 수 있는가? |
| 전환비용 | 고객이 경쟁 제품으로 이전하는 비용이 얼마나 높은가? |
| 네트워크 효과 | 사용자가 많을수록 제품이 더 좋아지는가? |
| 규모의 경제 | 규모에서 오는 원가 우위는 얼마나 큰가? |
| 기술/특허 장벽 | 기술이 몇 년 앞서 있는가? 복제될 수 있는가? |

해자 추세를 분석한다: 지난 5년간 넓어졌는가 좁아졌는가? 향후 5년 전망은?

**버핏식 질문**: 10년 후에도 이 해자가 남아 있을까? 무엇이 이를 파괴할 수 있는가?

### 4단계: 역발상 사고와 리스크 목록 — 멍거의 "거꾸로 생각하기"

- "이 회사가 실패할 수 있는 모든 경로"를 나열한다 (표: 경로/확률/영향 정도)
- 역사적 유추: 역사상 비슷한 위치에 있던 기업을 찾아, 그 결말이 어땠는지 분석
- 학제간 분석: 네트워크효과 이론, 기술수용곡선, 경쟁 게임이론 등의 모델로 교차 검증
- 편향 자체 점검: 서사 편향, 앵커링 효과, 생존자 편향
- 매도측(공매도) 핵심 논거 수집

**멍거식 질문**: 나는 어디서 가장 틀리기 쉬운가? 똑똑한 사람들은 왜 이 회사를 사지 않거나 공매도하는가?

### 5단계: 경영진 평가 — 돤융핑의 "적합한 사람" + 버핏의 "경영진의 정직성"

- CEO/창업자 핵심 의사결정 복기 (표: 시점/결정/결과/평가)
- 자본배분 능력: R&D 투자수익률, M&A 성공률, 자사주매입 타이밍
- 주주 이익 정합성: 경영진 지분 보유, 보수 구조, 지분 매도 기록
- 조직 역량: 팀 안정성, 핵심 인재 리스크
- 기업 문화 특성

**돤융핑식 질문**: CEO가 은퇴한다면, 이 회사는 경쟁력을 유지할 수 있는가?

### 6단계: 산업과 문명 트렌드 — 리루의 "문명 진화 프레임워크"

- 속한 산업이 "문명 차원의 패러다임 전환"에 있는지 판단
- 역사적 기술 혁명 유추 (증기기관/전기/인터넷/AI)
- TAM 성장 곡선과 한계 분석
- 산업 가치 사슬에서 이 회사의 위치
- 기술 노선 리스크
- 고객/공급업체 집중도 분석

**리루식 질문**: 20년 후를 돌아볼 때, 이 회사는 "이 시대의 스탠더드 오일"인가 "반짝했던 3Com"인가?

### 7단계: 밸류에이션과 안전마진 — 버핏의 "내재가치" + 돤융핑의 "적정 가격"

- 현재 시장 가격(핵심 밸류에이션 지표 표) —— **반드시 도구로 검산**
- 역방향 DCF: 현재 주가는 어떤 성장 기대를 내포하고 있는가?
- 세 가지 시나리오 밸류에이션 —— **반드시 도구로 정밀 계산, 암산 금지**:
```bash
python3 ~/ai-berkshire/tools/financial_rigor.py three-scenario \
  --price {주가} --eps {EPS} --shares {총발행주식수(억)} \
  --growth {낙관성장률} {중립성장률} {비관성장률} \
  --pe {낙관PE} {중립PE} {비관PE} --years 3 --currency {통화}
```
- 자체 역사적 밸류에이션과 비교
- 동종업계 밸류에이션과 비교

**돤융핑식 질문**: 만약 주식시장이 내일부터 5년간 문을 닫는다면, 이 가격에 보유할 의향이 있는가?

## 출력 요건

1. 모든 분석은 반드시 데이터 근거가 있어야 하며 출처를 첨부한다
2. Markdown 표를 사용해 핵심 데이터를 제시한다
3. 각 모듈 끝에는 반드시 해당 거장의 "질문"이 있어야 한다
4. 최종 완성 보고서는 `~/[기업명]투자연구보고서.md`에 작성한다
5. 결론은 명확해야 하며, 매수/관망/회피 제안을 회피하지 않고 제시한다
6. 밸류에이션 부분은 반드시 구체적인 가격 구간을 제시해야 한다
7. **보고서 서두**에 반드시 "정보 풍부도 등급"(A/B/C)과 "AI 리서치 한계 선언"을 포함해야 한다
8. **보고서 말미**에 반드시 "AI 분석 신뢰도"와 "투자 확실성"을 구분해야 한다 — 전자는 자료량에 좌우되고, 후자는 사업 본질에 좌우된다. 독자에게 이 보고서의 어떤 결론이 충분한 데이터에 근거하고, 어떤 결론이 제한된 정보에 기반한 추론인지 명확히 알려야 한다
9. 기업이 C등급(정보 희소)에 해당하면, 보고서 말미에 반드시 "1차 검증이 필요한 질문 목록"을 나열해야 한다 — 독자에게 현장 조사, 제품 체험, 공급망 인터뷰 등을 통해 AI의 사각지대를 보완하도록 권장한다

## 데이터 표본 점검 (발행 승인 프로세스)

보고서 작성 후, **반드시** 데이터 표본 점검을 실행하며, 통과 후에만 발행할 수 있다:

**1단계 — 표본 점검 목록 추출 (15% 무작위 표본):**
```bash
python3 ~/ai-berkshire/tools/report_audit.py extract \
  --report <보고서 파일 경로>
```
JSON 템플릿을 출력하며, 각 항목에 `fetched_value`(작성 대기)를 포함한다.

**2단계 — 취득 데이터 검증:**
목록의 각 데이터 항목에 대해, `skills/financial-data.md` 규정에 따라 신뢰할 수 있는 출처에서 데이터를 취득
(미국주: macrotrends+stockanalysis; 홍콩주: aastocks+macrotrends; A주: 동방재부+거차오자쉰),
`fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2`에 기입한다.

**3단계 — 판정 출력:**
```bash
python3 ~/ai-berkshire/tools/report_audit.py verdict \
  --results '<작성 완료된 JSON>' \
  --report <보고서 파일명>
```

- **【발행 승인】**: 모든 표본 점검 항목 편차 ≤ 1% → 보고서 발행 가능
- **【반려】**: 어느 항목이라도 편차 > 1% → 해당 데이터 수정 후 재점검, 승인될 때까지 반복
